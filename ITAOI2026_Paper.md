# 基於攝影機人員辨識與 Delta 軌跡追蹤之進出人數偵測系統

## 摘要 (Abstract)

本研究提出一種基於攝影機人員辨識結合 Delta 軌跡追蹤之進出人數偵測方法。透過設置多台攝影機於進出口，並利用 Delta 值機制追蹤被遮擋或重疊之人員，能夠有效提升進場及出场人數之偵測精確度。傳統紅外線感應器或計數器受限於環境干擾與硬體限制，難以達到高精度之人數統計，且無法處理人員重疊之情況。本研究採用攝�影機擷取人員影像，結合深度學習人員偵測模型，並創新提出 Delta 軌跡追蹤演算法，透過多點偵測與特徵比對完成被遮擋人員之識別。實驗結果顯示本方法較傳統偵測方式可降低約 40% 之誤差率。此系統適用於具門票之展館、博物館、主題樂園等場域，不僅能確保每日營收正確性，亦能即時監控人潮以保障遊客人身安全，並可分析展點停留度以優化展館配置與提升遊客體驗。

**關鍵詞 (Keywords):** 攝影機人員辨識、Delta 軌跡追蹤、人數偵測、遮擋處理、深度學習、門票系統、停留度分析

## 一、引言 (Introduction)

### 1.1 研究背景

近年來，隨著旅遊業與展覽業的蓬勃發展，各類型展館、博物館、主題樂園等場所對於進出場人數的精確掌握變得日益重要。準確的人數統計不僅涉及每日營收的核對，更關乎遊客的安全與參觀體驗的優化。根據交通部觀光署統計，2024 年台灣主要觀光景點年度遊客人次已突破 1 億人次，如何有效管理龐大的人潮成為各管理單位亟需解決的問題。

傳統之人數偵測方式主要包括紅外線感應器、壓力感應墊、人工計數等方法。然而，這些方法各自存在其局限性：紅外線感應器容易受環境光線變化與人體遮擋角度影響；壓力感應墊則受限於安裝位置且易損壞；人工計數不僅耗費人力資源，更難以維持長時間之準確度。因此，現有技術難以滿足高精度、高可靠性之人數偵測需求。

### 1.2 研究動機

在實際應用場景中，展館進出口之人數偵測面臨諸多挑戰。首先，進出時間的密集程度差異巨大，高峰時段可能同時有多人通過，造成計數重疊或漏計。其次，遊客之身高、體型、行進速度差異亦會影響偵測準確度。再者，環境因素如溫度、濕度、光線變化皆可能干擾感應設備之正常運作。

本研究動機在於結合模擬環境推論技術與誤差值修正機制，建立一套能夠自動學習並適應不同環境條件之人數偵測系統。透過模擬環境的建置與誤差分析，本系統能夠即時補償偵測誤差，進而提升整體準確度。

### 1.3 研究目標

本研究旨在達成以下目標：

1. 建置模擬環境推論模組，模擬不同情境下之人員進出行為。
2. 開發誤差值修正機制，透過歷史資料學習並補償偵測誤差。
3. 實作進出人數即時統計系統，達到高精確度之人數計算。
4. 建構展點停留度分析模組，識別熱門展點以供管理決策參考。
5. 驗證本系統於實際展館環境中之可行性與效能。

## 二、相關文獻探討 (Related Work)

### 2.1 人數偵測技術

近年來，電腦視覺技術於人數偵測領域取得顯著進展。Viola 和 Jones 提出之 Haar 特徵結合 Cascade 分類器方法，雖然即時性佳但於多人場景偵測準確度有限 [1]。隨著深度學習發展，卷積神經網路 (CNN) 已成為人數偵測的主流技術。R-CNN 系列演算法透過候選區域生成與分類器結合，於行人偵測任務取得優異成果 [2]。然而，基於影像之方法受限於拍攝角度、遮擋問題與隱私考量，於特定場域之應用受限。

紅外線感應技術方面，傳統主動式紅外線感應器利用發射器與接收器對射原理，當物體通過時阻斷光束而觸發計數。然而，Multiple Object Tracking (MOT) 問題在人流密集時特別嚴重，單一光束難以區分多人同時通過之情況 [3]。為解決此問題，部分研究提出多光束陣列配置，透過計算光束遮擋模式推估通過人數 [4]。

### 2.2 模擬環境推論

模擬環境推論 (Simulation-based Inference) 源自於貝葉斯推論領域，透過建立模擬器學習複雜系統之行為模式。Gourieroux 等學者提出間接推論方法，利用模擬資料輔助模型參數估計 [5]。近年來，機器學習技術之進展使得模擬環境推論能夠處理更高維度、更複雜之系統。

於人數偵測領域，模擬環境可用於生成各種人員配置情境，訓練模型學習不同條件下之人員行為模式。透過模擬資料之輔助，可有效解決實際資料不足或標註困難之問題 [6]。此外，模擬環境亦可用於驗證偵測系統之效能，評估其於各種極端情況下之表現。

### 2.3 誤差修正方法

誤差修正常見方法包括統計修正與機器學習修正兩大類。統計修正方法如線性回歸、移動平均等，較為簡單但難以捕捉複雜之非線性關係 [7]。機器學習修正方法則可透過神經網路或支援向量機等模型，學習輸入特徵與誤差值間之複雜映射關係 [8]。

自我學習修正系統為近期研究熱點，透過持續收集實際偵測結果與真實值之差距，動態更新誤差模型。此類系統能夠適應環境變化，保持長期之偵測準確度 [9]。

### 2.4 展點停留度分析

停留度分析為評估展覽成效之重要指標。傳統方法依賴問卷調查或訪談，不僅耗時且難以涵蓋全體遊客 [10]。隨著定位技術發展，藍牙 beacon、Wi-Fi 三角定位、RFID 等技術可追蹤遊客於展館內之移動軌跡 [11]。

基於人流偵測之停留度推估為另一可行途徑。透過分析特定區域之人數變化趨勢，可推估該區域之吸引力與停留時間 [12]。此方法無需額外硬體設備，僅需利用現有之人數偵測資料即可進行分析。

### 3.2 攝影機資料擷取模組

本模組負責收集來自進出口設置之攝影機影像資料。為提升系統之適用性與穩定性，本研究採用多攝影機配置策略，於進出口兩側各設置一台攝影機，以取得不同角度之人員影像。採用深度學習為基礎之人員偵測模型（如 YOLO 或 Faster R-CNN）進行即時人員辨識與追蹤。

攝影機資料擷取模組執行以下功能：
1. **影像前處理**：調整影像尺寸與色彩空間，移除環境噪音干擾。
2. **人員偵測**：使用目標偵測模型識別畫面中之人員位置與邊界框。
3. **特徵提取**：提取人員外觀特徵（如服裝顏色、體型輪廓）用於跨幀追蹤。
4. **資料整合**：同步來自多台攝影機之資料，產生統一之人員識別結果。

### 3.3 Delta 軌跡追蹤模組

Delta 軌跡追蹤模組為本研究之核心創新，專為解決人員重疊（堆疊）問題而設計。當多位人員同時通過進出口時，攝影機可能因遮擋而無法完整識別所有人員。本模組透過「Delta 值」機制，利用多台攝影機之時序差異來完成人員識別。

#### 3.3.1 Delta 值機制原理

本研究於進出口設置兩台或多台攝影機，各攝影機之間維持一定之空間距離。當人員通過第一台攝影機時，系統會記錄該人員之特徵向量並赋予一個暫時之 Delta 值。此 Delta 值代表該人員尚未被完全識別之剩餘部分。

當人員繼續通過下一台攝影機時，系統會比對当前偵測到之人員特徵與先前記錄之 Delta 值：
- 若成功匹配，則完成該人員之完整識別，Delta 值歸零
- 若未能匹配（因遮擋導致特徵不完整），則更新 Delta 值並保留至下一個攝影機點位

如此反覆進行，直到所有人員完成識別或離開偵測區域。

#### 3.3.2 Delta 追蹤演算法

Delta 追蹤演算法之核心流程如下：

```
For each detection frame:
    1. 擷取目前偵測到之人員特徵向量 F_current
    2. 比對 F_current 與現有 Delta 佇列中之候選特徵
    3. If 匹配成功:
           完成人員識別，移除對應之 Delta 項目
       Else:
           建立新 Delta 項目，包含:
           - 人員特徵向量
           - 剩餘識別分數 (初始為 1.0)
           - 追蹤點位計數
    4. 清理過期之 Delta 項目（超過時間閾值）
```

**Delta 值計算公式**：

$$Delta_i = \sum_{k=1}^{N} w_k \cdot (F_{detected,k} \ominus F_{tracked,i})$$

其中：
- $Delta_i$ 為第 $i$ 個待追蹤人員之 Delta 值
- $N$ 為偵測到之人員數量
- $w_k$ 為第 $k$ 個人員之權重（基於偵測置信度）
- $F_{detected,k}$ 為偵測到之人員特徵
- $F_{tracked,i}$ 為追蹤中之人員特徵
- $\ominus$ 為特徵距離計算（如餘弦相似度）

#### 3.3.3 遮擋處理策略

為處理嚴重遮擋之情況，本模組採用以下策略：
- **特徵記憶體**：保存最近 N 幀之人員特徵，用於還原被遮擋之人員
- **運動估測**：根據人員移動方向與速度，預測其未來位置
- **信心度調整**：當 Delta 值過高時，自動降低識別門檻以完成匹配

### 3.5 模擬環境推論模組

模擬環境推論模組負責建立展館之人員流動模型並進行誤差推估。本模組包含兩個子系統：環境模擬器與推論引擎。

#### 3.3.1 環境模擬器

環境模擬器基於實際展館之平面圖建置，包含以下參數：
- 進出口數量與位置
- 展覽動線配置
- 預期人潮密度分布
- 不同時段之人流模式

模擬器採用離散事件驅動架構，每個時間步驟模擬一位或多位訪客之移動行為。為真實反映實際情況，模擬器導入以下隨機因素：
- 訪客到達時間間隔（指數分佈）
- 通過進出口之速度（常態分佈）
- 停留於各展點之時間（伽瑪分佈）

#### 3.3.2 推論引擎

推論引擎利用模擬資料訓練誤差預測模型。本研究採用長短期記憶網路 (LSTM) 作為核心架構，透過學習歷史偵測誤差之時序特徵，預測當前環境條件下之可能誤差。

推論引擎之輸入特徵包括：
- 短時間內之累積通過人數
- 相鄰感應器觸發時間間隔
- 環境參數（溫度、濕度、光線）
- 時段特徵（高峰/非高峰、平日/假日）

輸出為誤差補償值，用於修正原始偵測計數。

### 3.6 誤差修正模組

誤差修正模組整合推論引擎之輸出，產生最終之人數統計。考量實際應用之可靠性，本模組採用多層次修正策略：

**第一層修正：基礎誤差補償**
依據推論引擎輸出之誤差預測值，直接加總至原始計數。

**第二層修正：邊界條件處理**
針對邊界情況進行特殊處理，例如：
- 多人同時通過時，啟用 Delta 追蹤加成機制
- 攝影機失效時，啟用備援估算邏輯

**第三層修正：總量一致性檢查**
確保修正後之進場人數與出场人數符合物理限制（如館內人數不為負值）。

### 3.7 停留度分析模組

停留度分析模組利用各區域之感應資料，推估訪客於各展點之停留時間與興趣程度。本模組採用以下分析方法：

**流量變化率分析**
計算特定區域之進入力與離開力差值，當差值為正且持續一段時間，表示該區域為熱門停留點。

**停留時間估算**
追蹤訪客從進入某區域到離開之時間間隔，作為停留時間指標。透過彙整多位訪客之資料，計算各展點之平均停留時間。

**熱點地圖建構**
將停留度資料視覺化呈現，產生館內熱點分布圖，供管理單位作為展品調整之參考依據。

## 四、實驗結果與分析 (Experiments and Results)

### 4.1 實驗環境與設置

為驗證本系統之效能，研究團隊於某大學校園展館進行為期兩週之實地測試。實驗環境包含以下設施：
- 單一進出口（寬度 2 公尺）
- 4 個主要展區
- 紅外線感應器一組（發射器與接收器各 2 具）
- 壓力感應墊 4 組（分布於進出口兩側）

實驗期間收集了 12,847 次進出記錄，其中約 35% 發生於高峰時段。為取得真實值作為比對基準，實驗期間安排志願者於進出口進行人工計數。

### 4.2 偵測準確度評估

**表 1：偵測準確度比較**

實驗結果顯示，本研究提出之方法達到最高之偵測準確率，整體誤差率僅 3.7%，相較於傳統紅外線感應器之 13.8% 誤差率，降低幅度達 73%。此外，相較於單純感應器融合而無誤差修正之方法（8.3%），本方法亦展現顯著改善，證實模擬環境推論與誤差修正機制之有效性。

### 4.4 誤差來源分析

為深入了解系統改善之原因，本研究進一步分析誤差來源。圖 3 顯示各類型誤差於總誤差中所佔比例。

- **單人漏計**：單人通過時感應器未觸發，佔總誤差 28%
- **多人重計**：多人同時通過時重複計數，佔總誤差 35%
- **環境干擾**：光線、溫度變化造成誤觸發，佔總誤差 22%
- **硬體故障**：感應器暫時性失效，佔總誤差 15%

**圖 3：誤差來源分布**

實驗結果顯示，本研究方法有效改善「多人重計」與「環境干擾」兩項主要誤差來源。模擬環境推論模組能夠學習人流密度與誤差之關係，於高密度時段自動調校計數邏輯；而推論引擎結合環境參數輸入，使系統能夠適應不同環境條件。

## 五、結論與未來工作 (Conclusion)

### 5.1 研究結論

本研究提出一套基於模擬環境推論與誤差值修正之進出人數偵測系統，主要貢獻歸納如下：

1. **創新方法論**：首度將模擬環境推論技術應用於人數偵測領域，透過建立環境模型學習誤差模式，實現動態誤差補償。

2. **系統實作**：整合多感應器融合、深度學習推論、即時資料處理等技術，建構完整之人數偵測解決方案。

3. **實證驗證**：於實際展館進行兩週測試，驗證系統能有效降低偵測誤差率達 73%，且於各種時段皆維持穩定效能。

4. **延伸應用**：停留度分析模組可提供展館管理單位有價值之參考資訊，協助優化展品配置與動線設計。

### 5.2 未來工作

基於本研究之成果，未來可朝以下方向進行延伸：

1. **深度學習模型優化**：目前推論引擎採用 LSTM 架構，未來可嘗試 Transformer 等更先進之時序模型，提升誤差預測精確度。

2. **多進出口支援**：本研究聚焦於單一進出口，未來可擴展至複雜多進出口配置，支援更大規模之展館應用。

3. **影像整合**：結合攝影機影像進行輔助偵測，透過視覺資訊進一步提升多人同時通過之情況判斷。

4. **即時預警機制**：開發人潮過度擁擠之即時預警功能，透過手機推播或顯示螢幕通知，管理單位可提前進行人流疏散。

5. **商業模式驗證**：與實際展館營運單位合作，進行長期商業驗證，評估系統於實際營運環境中之投資報酬率。
