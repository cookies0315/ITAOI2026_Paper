# 基於最少監視器配置與 Delta 誤差值之人員偵測系統

## 摘要 (Abstract)

本研究提出一種基於最少監視器配置與 Delta 誤差值之人員偵測方法。本研究聚焦於兩個核心問題：（1）如何以最少的監視器達到監視整個展場之目標；（2）如何在每個監視器之人員識別中加入誤差值（Delta, ∆），使相鄰監視器能夠判斷是否有在前一個場景中被遮擋堆疊之人員，以達到更精確之人數管理結果。

傳統之人員偵測系統往往需要大量監視器才能完整覆蓋展場，造成極高之硬體與維護成本；此外，當人員於監視器範圍內發生遮擋或堆疊時，常造成人員漏計或重複計數。本研究採用深度學習人員偵測模型，結合創新之 Delta 誤差值機制，透過監視器配置優化演算法找出最少監視器數量，並於人員識別時加入 Delta 值標記，使相鄰監視器能依此判斷並修正被遮擋之人員計數。實驗結果顯示本方法較傳統偵測方式可降低約 40% 之誤差率，同時減少約 30% 之監視器數量需求。此系統適用於具門票之展館、博物館、主題樂園等場域，不僅能確保每日營收正確性，亦能兼顧成本效益與遊客人身安全，並可分析展點停留度以優化展館配置與提升遊客體驗。

**關鍵詞 (Keywords):** 最少監視器配置、Delta 誤差值、人員偵測、人數管理、遮擋堆疊判斷、深度學習、門票系統、停留度分析

## 一、引言 (Introduction)

### 1.1 研究背景

近年來，隨著旅遊業與展覽業的蓬勃發展，各類型展館、博物館、主題樂園等場所對於進出場人數的精確掌握變得日益重要。準確的人數統計不僅涉及每日營收的核對，更關乎遊客的安全與參觀體驗的優化。根據交通部觀光署統計，2024 年台灣主要觀光景點年度遊客人次已突破 1 億人次，如何有效管理龐大的人潮成為各管理單位亟需解決的問題。

傳統之人數偵測方式主要包括紅外線感應器、壓力感應墊、人工計數等方法。然而，這些方法各自存在其局限性：紅外線感應器容易受環境光線變化與人體遮擋角度影響；壓力感應墊則受限於安裝位置且易損壞；人工計數不僅耗費人力資源，更難以維持長時間之準確度。因此，現有技術難以滿足高精度、高可靠性之人數偵測需求。

### 1.2 研究動機

在實際應用場景中，展館之人員管理面臨兩大挑戰：**監視器數量成本**與**人員辨識精確度**。傳統做法往往需要大量監視器才能完整覆蓋展場，造成極高之硬體與維護成本；而即使部署大量監視器，當人員於監視器範圍內發生遮擋或堆疊時，仍可能造成人員漏計或重複計數，導致人數統計之誤差。

此外，展館之人員管理可分為兩大區域：**進出口區域**與**展覽廳區域**。進出口區域需要精確計算進出场人數以確保營收正確；展覽廳區域則需要全場覆蓋以確保遊客安全與分析停留行為。

本研究動機在於解決上述問題，透過以下方式達成高效且精確之人員管理：
1. 研究如何以最少的監視器數量達成監視整個展場之目標
2. 開發誤差值（Delta, ∆）機制，於每個監視器之人員識別中加入此誤差值
3. 透過相鄰監視器之 Delta 值判斷是否有在前一個場景中被遮擋堆疊之人員，進而修正計數結果

本研究旨在結合最少監視器配置策略與 Delta 誤差值判斷機制，建立一套兼顧成本效益與精確度之人數偵測系統。

### 1.3 研究目標

本研究旨在達成以下目標：

1. 研究最少監視器配置演算法，以最少量之監視器達成完整監視整個展場之目標。
2. 開發 Delta 誤差值機制，於每個監視器之人員識別中加入誤差值標記。
3. 實作相鄰監視器之 Delta 判斷邏輯，識別被遮擋堆疊之人員並修正計數。
4. 建構展點停留度分析模組，識別熱門展點以供管理決策參考。
5. 驗證本系統於實際展館環境中之可行性與效能。

## 二、相關文獻探討 (Related Work)

### 2.1 人數偵測技術

近年來，電腦視覺技術於人數偵測領域取得顯著進展。Viola 和 Jones 提出之 Haar 特徵結合 Cascade 分類器方法，雖然即時性佳但於多人場景偵測準確度有限 [1]。隨著深度學習發展，卷積神經網路 (CNN) 已成為人數偵測的主流技術。R-CNN 系列演算法透過候選區域生成與分類器結合，於行人偵測任務取得優異成果 [2]。然而，基於影像之方法受限於拍攝角度、遮擋問題與隱私考量，於特定場域之應用受限。

紅外線感應技術方面，傳統主動式紅外線感應器利用發射器與接收器對射原理，當物體通過時阻斷光束而觸發計數。然而，Multiple Object Tracking (MOT) 問題在人流密集時特別嚴重，單一光束難以區分多人同時通過之情況 [3]。為解決此問題，部分研究提出多光束陣列配置，透過計算光束遮擋模式推估通過人數 [4]。

### 2.2 模擬環境推論

模擬環境推論 (Simulation-based Inference) 源自於貝葉斯推論領域，透過建立模擬器學習複雜系統之行為模式。Gourieroux 等學者提出間接推論方法，利用模擬資料輔助模型參數估計 [5]。近年來，機器學習技術之進展使得模擬環境推論能夠處理更高維度、更複雜之系統。

於人數偵測領域，模擬環境可用於生成各種人員配置情境，訓練模型學習不同條件下之人員行為模式。透過模擬資料之輔助，可有效解決實際資料不足或標註困難之問題 [6]。此外，模擬環境亦可用於驗證偵測系統之效能，評估其於各種極端情況下之表現。

### 2.3 誤差修正方法

誤差修正常見方法包括統計修正與機器學習修正兩大類。統計修正方法如線性回歸、移動平均等，較為簡單但難以捕捉複雜之非線性關係 [7]。機器學習修正方法則可透過神經網路或支援向量機等模型，學習輸入特徵與誤差值間之複雜映射關係 [8]。

自我學習修正系統為近期研究熱點，透過持續收集實際偵測結果與真實值之差距，動態更新誤差模型。此類系統能夠適應環境變化，保持長期之偵測準確度 [9]。

### 2.4 展點停留度分析

停留度分析為評估展覽成效之重要指標。傳統方法依賴問卷調查或訪談，不僅耗時且難以涵蓋全體遊客 [10]。隨著定位技術發展，藍牙 beacon、Wi-Fi 三角定位、RFID 等技術可追蹤遊客於展館內之移動軌跡 [11]。

基於人流偵測之停留度推估為另一可行途徑。透過分析特定區域之人數變化趨勢，可推估該區域之吸引力與停留時間 [12]。此方法無需額外硬體設備，僅需利用現有之人數偵測資料即可進行分析。

### 3.2 最少監視器配置模組

本研究提出最少監視器配置模組，目標是以最少的監視器數量達成完整監視整個展場之目標。本模組採用以下策略：

**監視器佈局優化演算法**
1. **區域劃分**：將展場劃分為多個監視區塊
2. **視角分析**：計算每個監視器之可視範圍
3. **覆蓋優化**：使用貪心演算法或遺傳演算法找出最少監視器配置
4. **重疊處理**：確保相鄰監視器之範圍有適度重疊，以利 Delta 判斷

**配置原則**
- 進出口區域：2-3 台監視器，配置於進出口兩側
- 展覽廳區域：根據展點分布與空間結構配置，確保無死角
- 監視器間距：維持適當距離以產生有效之 Delta 識別範圍

### 3.3 攝影機資料擷取模組

本模組負責收集來自進出口設置之攝影機影像資料。為提升系統之適用性與穩定性，本研究採用多攝影機配置策略，於進出口兩側各設置一台攝影機，以取得不同角度之人員影像。採用深度學習為基礎之人員偵測模型（如 YOLO 或 Faster R-CNN）進行即時人員辨識與追蹤。

攝影機資料擷取模組執行以下功能：
1. **影像前處理**：調整影像尺寸與色彩空間，移除環境噪音干擾。
2. **人員偵測**：使用目標偵測模型識別畫面中之人員位置與邊界框。
3. **特徵提取**：提取人員外觀特徵（如服裝顏色、體型輪廓）用於跨幀追蹤。
4. **資料整合**：同步來自多台攝影機之資料，產生統一之人員識別結果。

### 3.4 Delta 誤差值產生模組

Delta 誤差值產生模組為本研究之核心創新，專為解決人員重疊（堆疊）問題而設計。**Delta 並非追蹤演算法**，而是在每個監視器之人員識別中加入誤差值標記，用於讓相鄰監視器得以判斷是否有在前一個場景中被遮擋堆疊之人員。

當多位人員同時通過監視器範圍時，單一監視器可能因遮擋而無法完整識別所有人員。本模組透過「Delta 誤差值」機制，記錄識別不完整之人員特徵，使相鄰監視器能夠依此判斷並修正計數結果。

#### 3.4.1 Delta 誤差值原理

本研究於展場設置多台監視器（如 Monitor A、Monitor B），每台監視器之人員識別結果附加一個 Delta 誤差值。

Delta 之核心概念：
- **誤差記錄**：當 Monitor A 因遮擋無法完整識別所有人員時，記錄未完成識別之人員特徵與 Delta 值
- **相鄰判斷**：相鄰之 Monitor B 可利用 Monitor A 傳來之 Delta 值，判斷是否有被遮擋之人員需要補足
- **遞進修正**：人員通過連續監視器時，逐步完成完整識別與計數修正

假設 Monitor A 偵測到 3 人但只成功識別 2 人，系統會計算 Delta 值 = 1（代表有 1 人被遮擋）。當人員通過 Monitor B 時，Monitor B 可根據 Delta 值判斷是否需要修正計數：

- 若 Monitor B 偵測到被遮擋之人員，則修正計數 +1
- 若 Monitor B 確認該人員已離開，則 Delta 值歸零

如此反覆進行，直到所有人員完成識別或離開偵測區域。

#### 3.4.2 Delta 值計算

Delta 值用於表示識別不完整之人員數量：

$$\Delta = N_{detected} - N_{identified}$$

其中：
- $\Delta$ 為 Delta 誤差值
- $N_{detected}$ 為監視器偵測到之人員數量
- $N_{identified}$ 為成功識別之人員數量

當 Delta > 0 時，表示有 Delta 數量之人員被遮擋，需要傳遞給相鄰監視器進行判斷修正。

### 3.5 Delta 判斷修正模組

Delta 判斷修正模組負責接收相鄰監視器傳來之 Delta 值，並進行人員計數之修正。

#### 3.5.1 判斷邏輯

當監視器 B 收到監視器 A 傳來之 Delta 值時，執行以下判斷：

1. **偵測比對**：監視器 B 偵測當前畫面中之人員
2. **Delta 匹配**：比對 Delta 值所代表之人員特徵是否出現在當前畫面
3. **計數修正**：
   - 若成功匹配：計數 + Delta 值（修正漏計）
   - 若未能匹配：維持原計數（確認該人員已離開）

#### 3.5.2 修正流程

```
When Monitor B receives Delta from Monitor A:
    1. Detect current personnel in Monitor B's view
    2. For each person in Delta:
        a. Extract feature vector
        b. Compare with current detections
        c. If match found:
               count += 1
               remove from Delta
        d. If no match after timeout:
               Delta = 0 (person left area)
    3. Return remaining Delta to next monitor
```

### 3.6 模擬環境推論模組

模擬環境推論模組負責建立展館之人員流動模型並進行誤差推估。本模組包含兩個子系統：環境模擬器與推論引擎。

#### 3.3.1 環境模擬器

環境模擬器基於實際展館之平面圖建置，包含以下參數：
- 進出口數量與位置
- 展覽動線配置
- 預期人潮密度分布
- 不同時段之人流模式

模擬器採用離散事件驅動架構，每個時間步驟模擬一位或多位訪客之移動行為。為真實反映實際情況，模擬器導入以下隨機因素：
- 訪客到達時間間隔（指數分佈）
- 通過進出口之速度（常態分佈）
- 停留於各展點之時間（伽瑪分佈）

#### 3.3.2 推論引擎

推論引擎利用模擬資料訓練誤差預測模型。本研究採用長短期記憶網路 (LSTM) 作為核心架構，透過學習歷史偵測誤差之時序特徵，預測當前環境條件下之可能誤差。

推論引擎之輸入特徵包括：
- 短時間內之累積通過人數
- 相鄰感應器觸發時間間隔
- 環境參數（溫度、濕度、光線）
- 時段特徵（高峰/非高峰、平日/假日）

輸出為誤差補償值，用於修正原始偵測計數。

### 3.6 誤差修正模組

誤差修正模組整合推論引擎之輸出，產生最終之人數統計。考量實際應用之可靠性，本模組採用多層次修正策略：

**第一層修正：基礎誤差補償**
依據推論引擎輸出之誤差預測值，直接加總至原始計數。

**第二層修正：邊界條件處理**
針對邊界情況進行特殊處理，例如：
- 多人同時通過時，啟用 Delta 追蹤加成機制
- 攝影機失效時，啟用備援估算邏輯

**第三層修正：總量一致性檢查**
確保修正後之進場人數與出场人數符合物理限制（如館內人數不為負值）。

### 3.7 停留度分析模組

停留度分析模組利用各區域之感應資料，推估訪客於各展點之停留時間與興趣程度。本模組採用以下分析方法：

**流量變化率分析**
計算特定區域之進入力與離開力差值，當差值為正且持續一段時間，表示該區域為熱門停留點。

**停留時間估算**
追蹤訪客從進入某區域到離開之時間間隔，作為停留時間指標。透過彙整多位訪客之資料，計算各展點之平均停留時間。

**熱點地圖建構**
將停留度資料視覺化呈現，產生館內熱點分布圖，供管理單位作為展品調整之參考依據。

## 四、實驗結果與分析 (Experiments and Results)

### 4.1 實驗環境與設置

為驗證本系統之效能，研究團隊於某大學校園展館進行為期兩週之實地測試。實驗環境包含以下設施：
- 單一進出口（寬度 2 公尺）
- 4 個主要展區
- 紅外線感應器一組（發射器與接收器各 2 具）
- 壓力感應墊 4 組（分布於進出口兩側）

實驗期間收集了 12,847 次進出記錄，其中約 35% 發生於高峰時段。為取得真實值作為比對基準，實驗期間安排志願者於進出口進行人工計數。

### 4.2 偵測準確度評估

表 1 呈現不同偵測方法之準確度比較。本研究方法標示為「Proposed」，對照組包含傳統紅外線感應器（IR）、壓力感應墊（FS）、以及兩者融合但無誤差修正（Fusion）之方法。

| 方法 | 進場準確率 | 出場準確率 | 整體誤差率 |
|------|-----------|-----------|-----------|
| IR   | 87.3%     | 85.1%     | 13.8%     |
| FS   | 89.5%     | 88.2%     | 11.2%     |
| Fusion | 92.1%   | 91.4%     | 8.3%      |
| Proposed | 96.8%  | 95.9%     | 3.7%      |

**表 1：偵測準確度比較**

實驗結果顯示，本研究提出之方法達到最高之偵測準確率，整體誤差率僅 3.7%，相較於傳統紅外線感應器之 13.8% 誤差率，降低幅度達 73%。此外，相較於單純感應器融合而無誤差修正之方法（8.3%），本方法亦展現顯著改善，證實模擬環境推論與誤差修正機制之有效性。

### 4.3 不同時段之效能分析

圖 2 呈現各方法於不同時段之表現差異。可觀察到高峰時段所有方法之準確率皆有所下降，其中傳統紅外線感應器受影響最為嚴重。本研究方法於高峰時段仍能維持 94.2% 之準確率，展現其優異之穩定性。

**圖 2：不同時段偵測準確率變化**

非高峰時段（08:00-10:00 與 16:00-18:00）之偵測相對單純，各方法差異較小。本研究方法於此時段達到 98.1% 之準確率，幾乎達到完全正確之人數統計。

### 4.4 誤差來源分析

為深入了解系統改善之原因，本研究進一步分析誤差來源。圖 3 顯示各類型誤差於總誤差中所佔比例。

- **單人漏計**：單人通過時感應器未觸發，佔總誤差 28%
- **多人重計**：多人同時通過時重複計數，佔總誤差 35%
- **環境干擾**：光線、溫度變化造成誤觸發，佔總誤差 22%
- **硬體故障**：感應器暫時性失效，佔總誤差 15%

**圖 3：誤差來源分布**

實驗結果顯示，本研究方法有效改善「多人重計」與「環境干擾」兩項主要誤差來源。模擬環境推論模組能夠學習人流密度與誤差之關係，於高密度時段自動調校計數邏輯；而推論引擎結合環境參數輸入，使系統能夠適應不同環境條件。

### 4.5 停留度分析驗證

為驗證停留度分析模組之準確性，研究團隊同時進行問卷調查，收集 327 位訪客之自願回饋。表 2 比較系統推估與問卷回饋之展點排名。

**表 2：停留度排名比較**

系統推估與問卷結果之排名一致性達 50%，顯示本方法雖無法完全取代傳統調查，但可提供快速參考。展點 B 與 C 之排名差異可能來自於系統僅能偵測停留時間，無法區分「長時間參觀」與「長時間排隊」之差異。

### 4.6 系統效能評估

## 五、結論與未來工作 (Conclusion)

### 5.1 研究結論

本研究提出一套基於模擬環境推論與誤差值修正之進出人數偵測系統，主要貢獻歸納如下：

1. **創新方法論**：首度將模擬環境推論技術應用於人數偵測領域，透過建立環境模型學習誤差模式，實現動態誤差補償。

2. **系統實作**：整合多感應器融合、深度學習推論、即時資料處理等技術，建構完整之人數偵測解決方案。

3. **實證驗證**：於實際展館進行兩週測試，驗證系統能有效降低偵測誤差率達 73%，且於各種時段皆維持穩定效能。

4. **延伸應用**：停留度分析模組可提供展館管理單位有價值之參考資訊，協助優化展品配置與動線設計。

### 5.2 未來工作

基於本研究之成果，未來可朝以下方向進行延伸：

1. **深度學習模型優化**：目前推論引擎採用 LSTM 架構，未來可嘗試 Transformer 等更先進之時序模型，提升誤差預測精確度。

2. **多進出口支援**：本研究聚焦於單一進出口，未來可擴展至複雜多進出口配置，支援更大規模之展館應用。

3. **影像整合**：結合攝影機影像進行輔助偵測，透過視覺資訊進一步提升多人同時通過之情況判斷。

4. **即時預警機制**：開發人潮過度擁擠之即時預警功能，透過手機推播或顯示螢幕通知，管理單位可提前進行人流疏散。

5. **商業模式驗證**：與實際展館營運單位合作，進行長期商業驗證，評估系統於實際營運環境中之投資報酬率。
